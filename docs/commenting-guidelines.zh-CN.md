# 注释规范

本仓库的注释目标是让接口调用方和维护者快速看清业务边界、事务规则和集成约束，而不是堆叠说明性文字。

## C# 成员级注释

- 已纳入注释治理范围的 C# 文件，类、字段和方法都使用中文 XML 文档注释。
- 字段注释说明保存的状态、协作对象、配置来源、缓存内容或领域含义。
- 方法注释说明调用时机、职责、事务边界、关键副作用和失败语义。
- 本次不强制为属性、事件和枚举成员逐个补注释，但公共契约仍应保持清晰。

## Controller 与公共契约

- 控制器 Action、服务接口、SignalR 契约、共享 DTO 使用 XML 文档注释。
- 注释需要说明路由目的、入参约束、成功/失败语义，以及与前端或外部设备的交互场景。
- 公共契约只描述当前真实行为，不提前承诺未来逻辑。

## Service 实现

- 对事务边界、FIFO 选择、幂等控制、锁定生命周期、仓库隔离、回调处理、异常兜底等关键节点使用注释说明。
- 注释要解释设计原因，例如“避免跨仓库误释放”“事务内再次确认库位可用”“回调失败也要通知前端”。
- 即使成员需要注释，也不为简单的 LINQ、赋值、集合转换逐行写说明。

## 数据层与模型

- `DbContext` 只注释高价值信息，例如索引用途、唯一性约束、关键实体映射。
- 已纳入治理范围的模型字段按成员级规则补充注释，普通属性不强制逐项补齐。
- 配置或启动文件只补充最小必要说明，不写大段重复性说明。

## 禁止事项

- 不新增乱码、重复、失效或与实现不一致的注释。
- 不使用“这里调用接口”“这里返回结果”这类低信息量注释。
- 不把注释当需求文档；超出代码上下文的背景说明应放到独立文档中。
